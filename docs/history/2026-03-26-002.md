# Session 002 — 2026-03-26

## Summary
설계 문서 리뷰 피드백 반영. 5가지 우려사항 해결.

## Review Feedback & Resolution

### 1. OBS 컴퓨터 부하 문제
- **우려**: OBS 인코딩 + Node.js + API 스트리밍 동시 실행 시 부하
- **해결**: PoC-3으로 실제 측정 후 판단. CPU 70% 이상이면 NVENC 전환 또는 미니PC 분리
- **반영**: architecture.md에 "OBS 컴퓨터 부하 관리" 섹션 추가

### 2. 오디오 캡처 방식 구체화
- **우려**: Node.js에서 Windows 시스템 오디오 캡처의 불안정성
- **해결**: SoX (1순위) → FFmpeg (fallback) → portaudio (비추)
- **반영**: Audio Capture Module에 구체적 커맨드 + PoC-1 추가

### 3. TTS 오디오 큐 전략
- **우려**: 이전 TTS 재생 중 다음 번역 도착 시 처리 방법 없음
- **해결**: 큐 기반 순차 재생 + 3개 이상 밀리면 스킵(텍스트만 표시)
- **반영**: "TTS Audio Queue" 섹션 신규 추가, seq number 프로토콜

### 4. API 장애 fallback
- **우려**: 3개 외부 API 직렬 호출 시 하나 실패하면 전체 중단
- **해결**: 단계별 graceful degradation (STT→번역→TTS 각각 독립 실패 처리)
- **반영**: "API Fallback 전략" 섹션 추가, 모든 API에 5초 타임아웃

### 5. 노이즈 제거 구현 방법
- **우려**: P0 요구사항인데 아키텍처에 구현 방법 없음
- **해결**: 3단계 (OBS 노이즈 게이트 → Azure STT 내성 → 후처리 감탄사 필터)
- **반영**: "Noise Filtering" 섹션 신규 추가
